# 📖 Descrição

O **Back-End** é a parte invisível de um sistema. Nele são aplicadas
todas as funcionalidades dos programas, ou seja, tudo que acontece por
trás da interface que o usuário vê (**Front-End**).

------------------------------------------------------------------------

# 🧠 Principais termos do Back-End

## 🔗 API (Application Programming Interface)

É a forma de comunicação entre sistemas.

## 🗄️ Database (Banco de Dados)

Onde os dados do sistema ficam armazenados.

## 🎮 Controller

Onde fica a lógica da aplicação.

## 🔄 ORM (Object Relational Mapping)

Ferramenta que conecta o código ao banco de dados.

## 🌐 HTTP Status Code

Código de resposta do servidor.

Principais códigos: - **200** → OK\
- **201** → Criado\
- **400** → Erro do cliente\
- **404** → Não encontrado\
- **500** → Erro no servidor

------------------------------------------------------------------------

# ⚙️ Funcionalidades do Back-End

O Back-End tem várias funções essenciais:

## 🧩 Processar dados

Recebe informações do usuário e processa.

## 📏 Regras de negócio

Define como o sistema funciona.

**Exemplos:** - Quem pode acessar o quê\
- Cálculo de preços\
- Validação de dados

## 💾 Comunicação com banco de dados

-   Salva informações\
-   Busca dados\
-   Atualiza registros

## 🔐 Segurança

-   Autenticação (login/senha)\
-   Autorização (permissões)\
-   Criptografia de dados

## 🔌 Integração com outros sistemas

-   APIs externas\
-   Pagamentos\
-   Serviços de terceiros

# 📌 Resumo++

-   Parte invisível do sistema (lógica e funcionamento)\
-   Processa dados e aplica regras de negócio\
-   Faz comunicação com banco de dados e outros sistemas\
-   Garante segurança (autenticação e autorização)\
-   Utiliza conceitos como API, Controller, ORM e HTTP\
-   Usa linguagens como Java, JavaScript, Python, PHP, .NET e SQL
