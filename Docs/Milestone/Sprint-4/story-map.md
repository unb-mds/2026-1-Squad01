# 🗺️ Story Map - Organização do Projeto

## 📌 Objetivo
Descrever e organizar as funcionalidades do sistema de forma visual e estruturada, priorizando a experiência do usuário e facilitando o planejamento das entregas.

---

## 👤 Persona / Usuário
- Usuário visitante do sistema  
- Desenvolvedor (admin/manutenção futura)  

---

## 🚀 Jornada do Usuário (Macro Fluxo)

1. Acessar o sistema  
2. Navegar pelas funcionalidades  
3. Consumir informações / executar ações  
4. Encerrar uso  

---

## 🧩 Atividades (Epics)

### 1. Acesso ao Sistema
- Visualizar página inicial  
- Carregar conteúdo principal  

### 2. Navegação
- Acessar menus  
- Navegar entre páginas  
- Buscar informações  

### 3. Interação
- Consumir dados da API  
- Visualizar listagens (ex: produtos, usuários)  
- Filtrar / pesquisar dados  

### 4. Sistema (Back-end/API)
- Criar endpoints  
- Processar requisições  
- Retornar respostas (JSON)  

---

## 📝 Tarefas (User Stories)

### 🔹 Acesso ao Sistema
- [ ] Como usuário, quero acessar a página inicial para visualizar o sistema  
- [ ] Como usuário, quero que a página carregue rapidamente  

### 🔹 Navegação
- [ ] Como usuário, quero navegar entre páginas facilmente  
- [ ] Como usuário, quero encontrar informações rapidamente  

### 🔹 Interação
- [ ] Como usuário, quero visualizar dados vindos da API  
- [ ] Como usuário, quero filtrar resultados  
- [ ] Como usuário, quero pesquisar informações específicas
